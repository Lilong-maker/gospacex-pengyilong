# 订单超时自动关单数据流图 (DFD)

## 系统概述

本文档描述电商系统中订单超时自动关单流程的数据流。当用户下单后未在规定时间内完成支付，系统将自动取消订单并回滚库存。

---

**外部实体:**
- **用户**: 下单并支付的客户端
- **支付网关**: 处理支付回调的第三方服务

**核心流程:**
- 订单创建 → 定时检查 → 超时关单 → 库存回滚

## 数据存储

| 存储 | 用途 | 数据内容 |
|------|------|----------|
| **MySQL** | 订单主数据 | 订单ID、用户ID、状态、金额、创建时间、支付时间 |
| **Redis** | 订单TTL缓存 | 订单ID → 过期时间映射，用于快速超时判断 |
| **Delay Queue** | 超时任务调度 | 订单ID、计划执行时间、任务类型 |

## 数据流说明

### 流1: 订单创建
- **来源**: 用户
- **目标**: 订单服务
- **数据**: `{order_id, user_id, goods_id, quantity, amount, status: "PENDING"}`
- **处理**: 创建订单记录，锁定库存，设置超时检查任务

### 流2: 库存锁定
- **来源**: 订单服务
- **目标**: 库存服务
- **数据**: `{order_id, goods_id, quantity, lock_type: "RESERVE"}`
- **处理**: 预扣库存，生成锁定记录

### 流3: 超时检查触发
- **来源**: 调度器 (Delay Queue)
- **目标**: 订单服务
- **数据**: `{order_id, task_type: "TIMEOUT_CHECK", trigger_time}`
- **处理**: 查询订单当前状态

### 流4: 状态验证
- **来源**: 订单服务
- **目标**: 订单数据库
- **数据**: 查询订单状态
- **结果**: 返回 `PAID` 或 `PENDING`

### 流5: 自动关单 (条件: 未支付)
- **来源**: 订单服务
- **目标**: 订单数据库
- **数据**: `{order_id, status: "CANCELLED", reason: "TIMEOUT", cancelled_at}`
- **处理**: 更新订单状态为已取消

### 流6: 库存回滚
- **来源**: 订单服务
- **目标**: 库存服务
- **数据**: `{order_id, goods_id, quantity, action: "RELEASE"}`
- **处理**: 释放锁定的库存，恢复可用库存

### 流7: 支付成功回调 (异步)
- **来源**: 支付网关
- **目标**: 订单服务
- **数据**: `{order_id, payment_id, status: "PAID", paid_at}`
- **处理**: 更新订单状态，取消超时检查任务

## 关键设计决策

### 1. 延迟队列选择
- 使用 Redis Sorted Set 或专业延迟队列 (如 RabbitMQ DLX)
- 订单ID作为成员，执行时间戳作为分数

### 2. 库存回滚策略
- **乐观锁**: 基于库存版本号防止并发问题
- **幂等性**: 回滚操作可重复执行，结果一致
- **补偿机制**: 回滚失败时记录日志，人工介入或定时重试

### 3. 分布式事务
- 采用 **Saga 模式**: 订单取消和库存回滚作为独立事务
- 失败重试: 库存回滚失败时延迟重试3次
- 最终一致性: 允许短暂不一致，最终达到一致状态

### 4. 幂等性保障
- 所有关单操作使用订单ID + 状态机保证幂等
- 库存回滚基于锁定记录ID，避免重复释放

---

## 异常处理

| 异常场景 | 处理策略 |
|----------|----------|
| 调度任务丢失 | 定期扫描PENDING订单，补充超时检查任务 |
| 库存回滚失败 | 记录失败日志，进入重试队列，告警通知 |
| 重复关单请求 | 基于状态机幂等处理，返回已取消结果 |
| 数据库不可用 | 缓存到Redis，恢复后异步处理 |

---

## 时序参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 订单超时时间 | 30分钟 | 从创建到自动取消的最大等待时间 |
| 调度检查间隔 | 1分钟 | 延迟队列轮询间隔 |
| 库存回滚重试 | 3次 | 回滚失败后的最大重试次数 |
| 重试间隔 | 5秒 | 每次重试之间的等待时间 |
